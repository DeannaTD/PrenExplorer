Урок 18. Работа с исключениями

В некоторых ситуациях мы знаем, что при выполнении кода могут возникнуть ошибки. Если эти исключения не обработать, то ваше приложение может “сломаться” -- то есть выдать критическую ошибку и закрыться. В таких участках мы используем конструкцию try/except/finally

Предположим, мы ожидаем, что в нашу (гипотетическую) программу, которая делит число 10 на переданное число, могут передать любую цифру, в том числе 0. При делении на ноль, Python выдает следующее исключение (exception):

passed_number = 0

our_magic_number = 42

result = our_magic_number / passed_number

Traceback (most recent call last):

  File "", line 1, in

ZeroDivisionError: division by zero

Когда такого рода ошибки происходят, программа, как уже говорилось, ломается. Мы “ловим” исключения таким образом:

passed_number = 0

our_magic_number = 42

result = None

try:

...      result = our_magic_number / passed_number

... except ZeroDivisionError:

...      result = 0

... finally:

...      print('Calculation is done')

... 

Calculation is done

>>>

Что здесь произошло? Мы прикинулись, что passed_number содержит число, которое передал пользователь (для упрощения примера). our_magic_number - число, которое “прошитое” в коде (прикинемся, что это очень важное число, которое помогает нам вычислить какие-то важные вещи). Конструкцию открывает ключевое слово try, далее идет двоеточие, которое традиционно открывает блок кода с отступом в четыре пробела. В данном блоке находится код, где потенциально может произойти “авария”, и внизу, после блока try идёт except, где мы указываем тип исключения, на который должен реагировать следующий блок. В нашем случае, мы реагируем на тип исключения “ZeroDivisionError”, и в качестве реакции в переменную result записываем 0, так как в теле try при попытке записать результат вычисления произошла ошибка, и в переменной result все еще хранится None (чего мы не хотим).

А код в блоке finally выполняется при любом исходе. В этом смысл этого блока -- выполнять код при любом исходе try/except.

Важно понимать, что при обработке исключения в блоке except также может быть выдано исключение, для обработки которой может быть добавлен еще один блок exception.
